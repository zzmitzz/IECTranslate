Setup Instructions

Prerequisites

  1. Ensure your WebRTC server is running (Python backend)
  2. Check that the server is accessible at the configured URL
  3. Make sure your browser supports WebRTC (Chrome, Firefox, Safari)
  4. Allow microphone access when prompted

  • HTTPS is required for production use (WebRTC requirement)
  • Local development works with HTTP on localhost
  • Ensure your firewall allows WebRTC connections
  • Check browser console for detailed connection logs

Server Configuration

Default server settings for testing:

Server URL: ws://localhost:8000/ws
Room ID: test-room
Peer ID: peer-1 (or any unique identifier)

To start the server, run:

python run_server.py

Testing Scenarios

Test the following scenarios to verify functionality:

  • Single Peer: Connect and test audio capture/playback
  • Multiple Peers: Open multiple browser tabs with different peer IDs
  • Audio Recording: Test recording functionality
  • Device Switching: Test different audio input devices
  • Connection Recovery: Test reconnection after disconnection

Troubleshooting

Common issues and solutions:

  • Connection Failed: Check server status and WebSocket URL
  • No Audio: Check microphone permissions and device selection
  • Poor Quality: Check network conditions and audio constraints
  • Recording Issues: Ensure browser supports MediaRecorder API

For detailed debugging, check the browser console and the application logs.
